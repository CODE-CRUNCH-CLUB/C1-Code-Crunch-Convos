# Week 3 — Challenges

Challenges are bigger than exercises. Where an exercise pins down one
specific idea, a challenge mixes several ideas in a setting that feels a
bit more like real software. Expect each one to take **45–90 minutes**.

You will write the code yourself in a `.py` file — the markdown specs
below give you the requirements, hints, and acceptance criteria. There
is no starter file: the act of building the file from scratch is part of
the practice.

## Workflow tips

1. **Read the entire spec first.** Resist the urge to start typing
   before you understand the acceptance criteria.
2. **Write the simplest version that runs.** A working ugly solution is
   miles better than a half-written elegant one.
3. **Then refactor.** Pull repeated logic into small helpers, rename
   confusing variables, add a docstring at the top of the file.
4. **Commit when it works.** A challenge that passes its acceptance
   criteria is worth a commit, even if you plan to polish it further.
