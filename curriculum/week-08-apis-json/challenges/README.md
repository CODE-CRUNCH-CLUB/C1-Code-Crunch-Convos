# Week 8 — Challenges

Two longer, open-ended problems. Both are write-and-think exercises: you submit working Python *plus* a short markdown explanation of the design choices you made.

Where the exercises were "do exactly this," challenges are "here is the goal — figure out how." That ambiguity is realistic. Almost every task you will face as a developer starts vague.

| # | File | Estimated time |
|---|---|---|
| 01 | `challenge-01-currency-converter.md` | ~2 hours |
| 02 | `challenge-02-github-user-stats.md` | ~3 hours |

---

## How to submit

For each challenge, create two files next to the markdown spec:

```
challenges/
    challenge-01-currency-converter.md     <- the spec (given)
    challenge-01-currency-converter.py     <- your code
    challenge-01-currency-converter-notes.md  <- your write-up
```

The write-up should answer, briefly (one short paragraph each):

1. What API endpoints did you call?
2. How did you handle errors?
3. What is one thing you would improve given another hour?

Honest, terse write-ups are better than long padded ones.

---

## Both challenges use FREE, no-key APIs.

You do not need to sign up for anything. If your code asks for a key, you took a wrong turn — re-read the spec.
