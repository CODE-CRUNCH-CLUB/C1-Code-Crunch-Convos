# Week 1 Challenges

Challenges are longer than exercises and ask you to combine multiple
concepts from the week. They are still beginner-friendly but expect you to
make a few of your own design decisions. Aim to spend 30–60 minutes on
each.

## Ground Rules

- Work inside an activated virtual environment.
- Commit your solution to the same Git repository as the mini-project (or
  a sibling repository — your choice).
- Each challenge has clear *Requirements* and *Stretch goals*. The
  requirements are the bar to clear; stretch goals are extra credit if you
  finish early.

## How to Submit (Self-Check)

There is no central submission for Week 1. For each challenge, verify
that:

1. Your script runs without errors from a fresh terminal.
2. You have at least one Git commit per challenge with a clear message.
3. The output matches the challenge's example output (approximately —
   exact spacing may vary).
