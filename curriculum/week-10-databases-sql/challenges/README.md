# Week 10 Challenges

Two bigger projects that connect this week's database skills back to systems you built in previous weeks. Pick **one** to do during the week. Pick the other if you want to come back and reinforce the material.

## The challenges

| #  | File                                | Builds on   | What you'll do                                                            |
|----|-------------------------------------|-------------|---------------------------------------------------------------------------|
| 1  | `challenge-01-library-with-sqlite.md` | Week 7 OOP | Port the JSON-backed library management system to SQLite.                |
| 2  | `challenge-02-blog-with-db.md`        | Week 9 Flask | Port the Flask blog to a SQLite-backed model (sqlite3 *or* SQLAlchemy). |

## Submission checklist

For either challenge, your repo should include:

- [ ] A clean schema in a top-level `schema.sql` (or generated by your Python code on first run).
- [ ] All database access through **parameterized queries** — no f-strings, no `%`, no `.format()` into SQL.
- [ ] A README explaining how to set up the DB and run the app.
- [ ] At least three meaningful tests (we'll cover pytest in Week 11, but you can write basic ones now using `assert`).
- [ ] No `.db` file committed to git. Add `*.db` to your `.gitignore`.

## Tips

- Start with the *schema*. Sketch the tables on paper before writing any code.
- Run the SQL by hand in the `sqlite3` CLI before wiring it into Python.
- Migrate gradually: get one feature working end-to-end (e.g., `add_book`) before refactoring the rest.
- Test on a *throwaway* `.db` file so you can wipe it freely.
