# Week 2 — Challenges

Challenges are longer than exercises and have less hand-holding. They're
designed to push you to combine multiple concepts and to make small
design decisions. Each one should take 45 to 90 minutes.

## How to Tackle a Challenge

1. Read the challenge specification all the way through before writing
   any code.
2. Sketch the program on paper or in comments at the top of a new file:
   what does it read, what does it compute, what does it print?
3. Implement and test by hand. Try several inputs, including edge
   cases.
4. Commit your solution to your Week 2 repository as
   ``challenges/challenge-XX-name.py``.

## Submission Pattern

Both challenges produce **runnable Python scripts**, not just markdown
notes. Save them inside the same ``challenges/`` directory as files
called ``challenge-01-tip-calculator.py`` and
``challenge-02-mad-libs.py``. The markdown file you're reading
describes *what to build*; the ``.py`` file is *your solution*.
