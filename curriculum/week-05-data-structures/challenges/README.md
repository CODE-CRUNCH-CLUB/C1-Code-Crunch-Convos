# Week 5 — Challenges

Two open-ended challenges that go beyond the guided exercises. Plan before you code. Read the markdown spec carefully, sketch the structure, then write the program.

| # | File | Topic |
|---|---|---|
| 01 | `challenge-01-anagram-groups.md` | Grouping with a dict |
| 02 | `challenge-02-inventory-tracker.md` | Nested dictionaries |

---

## How to submit

For each challenge, create a `solution.py` (or `solution-01.py`, `solution-02.py`) inside this folder, push it to your fork, and link it from your weekly write-up.

Each solution should:

- Run with `python solution-XX.py` and print sample output.
- Include type hints on every function.
- Include at least 5 `assert` statements covering the main cases.
- Have a one-paragraph reflection at the top (in a docstring) about *why* you chose the data structure you chose.

---

## Difficulty

| Challenge | Estimated time | Difficulty |
|---|---|---|
| 01 — Anagram groups | 60–90 min | Medium |
| 02 — Inventory tracker | 90–120 min | Medium-Hard |

Both challenges are solvable with **only** the concepts from this week — no extra modules required (though `collections.defaultdict` will make Challenge 01 much cleaner).
